# Platform Builder 游戏玩法分析

**游戏类型**: 模拟经营 + 基地建设
**灵感来源**: Metal Gear Solid V: Mother Base
**引擎**: Godot 4.6
**当前状态**: 原型完成

---

## 🎮 核心玩法概述

### 一句话描述
**建造海上平台基地，管理资源与员工，发送远征任务。**

---

## 🔄 核心循环 (Core Loop)

### 主循环
```
建造平台 → 生产资源 → 建造更多平台 → 解锁新功能 → 扩张基地
    ↑                                                          ↓
    └────────────── 长期目标：完成故事/达成成就 ←──────────┘
```

### 详细循环分解

#### 1. 资源采集循环
```
时间流逝 (每秒)
    ↓
平台自动生产
    ↓
获得 Materials (材料)
获得 Fuel (燃料)
    ↓
支付员工维护成本
```

#### 2. 平台建造循环
```
拥有足够资源
    ↓
选择建造槽位
    ↓
选择平台类型 (R&D/Combat/Support/Intel/Medical)
    ↓
检查建造条件
    ├─ 资源足够？ ✓
    ├─ 部门未满？ ✓
    └─ 父平台未满？ ✓
    ↓
建造成功
    ↓
获得新能力 (生产/床位/战斗力)
```

#### 3. 员工管理循环
```
拥有GMP & 床位
    ↓
招募员工 (50 GMP)
    ↓
新员工进入招募池
    ↓
分配到部门 (R&D/Combat/Support/Intel/Medical)
    ↓
获得部门加成
    ├─ R&D → 研究速度↑
    ├─ Combat → 战斗力↑
    ├─ Support → 床位↑
    ├─ Intel → 远征成功率↑
    └─ Medical → 床位↑
```

#### 4. 远征任务循环
```
积累足够战斗力
    ↓
选择任务 (难度匹配)
    ↓
发送远征
    ↓
等待完成 (60-120秒)
    ↓
获得奖励 (资源/GMP/员工)
    ├─ 成功 → 全部奖励
    ├─ 部分成功 → 50%奖励
    └─ 失败 → 无奖励 + 可能伤亡
```

## 🏗️ 核心机制分析

### 1. 平台建造系统

#### 树形扩展机制 (核心创新)
```
HQ (总部)
│
├─ R&D 平台 1
│  ├─ Support 平台 A
│  │  └─ Medical 平台 → 3级深度
│  └─ Combat 平台 B
│
├─ R&D 平台 2
└─ ... (理论上无限扩张)
```

**设计亮点**:
- ✅ 视觉上有趣 (海上基地增长)
- ✅ 策略选择 (先扩展哪个分支？)
- ✅ 有机成长 (基地看起来自然)
- ✅ 部门组合 (相邻平台有加成)

#### 限制机制
- **部门限制**: 每种类型最多6个
- **父平台限制**: 每个平台最多6个子平台
- **资源限制**: 建造需要Materials和Fuel
- **床位限制**: 员工需要床位

**平衡性分析**:
- 限制确保玩家需要策略选择
- 防止单一策略最优解
- 鼓励多样化发展

---

### 2. 资源系统

#### 资源类型
```
Materials (材料)
├─ 用途: 建造平台
├─ 来源: R&D (2/秒), Combat (1/秒), Medical (1/秒)

Fuel (燃料)
├─ 用途: 建造平台 + 远征
├─ 来源: Support (2/秒), Intel (1/秒), Combat (1/秒)

GMP (Global Money Points)
├─ 用途: 招募员工
├─ 来源: 远征奖励

Staff Count (员工数)
├─ 限制: 床位
├─ 影响: 维护成本

Beds (床位容量)
├─ 来源: HQ (5) + Support (5) + Medical (3)
└─ 限制: 最大员工数
```

#### 资源平衡分析
| 平台类型 | Materials/秒 | Fuel/秒 | 特殊功能 | 建造成本 |
|---------|-------------|---------|---------|---------|
| R&D | 2 | 0 | 研究加速 | 50材料 + 10燃料 |
| Support | 0 | 2 | 床位+5 | 30材料 + 40燃料 |
| Combat | 1 | 1 | 战斗力 | 40材料 + 30燃料 |
| Intel | 0 | 1 | 远征成功率 | 35材料 + 25燃料 |
| Medical | 1 | 0 | 床位+3 | 25材料 + 25燃料 |

**设计评价**:
- ✅ 相互依赖 (需要多种资源)
- ✅ 风险回报 (高产量=高成本)
- ✅ 策略选择 (优先发展哪种？)

---

### 3. 员工系统

#### 员工属性
```gdscript
Staff (员工)
├─ ID: 唯一标识
├─ 姓名: 随机生成
├─ 技能等级: 1-5星
├─ 专业: Combat/Research/Logistics/Medicine/Engineering/None
├─ 部门: 当前分配 (或未分配)
└─ 生产力: 技能 × 专业匹配 × 部门匹配
```

#### 部门加成机制
```
R&D 部门
├─ 加成: 研究速度 × (1 + 0.1 × 员工数 × 专业匹配)
└─ 作用: 加速科技树/任务 (如果实现)

Combat 部门
├─ 加成: 战斗力 + 0.5 × 员工数 × 专业匹配
└─ 作用: 解锁高难度远征

Support 部门
├─ 加成: 床位 +5 (固定) + 员工生产力
└─ 作用: 扩大规模上限

Intel 部门
├─ 加成: 远征成功率 + 5% × 员工数
└─ 作用: 提高远征效率

Medical 部门
├─ 加成: 床位 +3 (固定) + 员工生产力
└─ 作用: 扩大规模上限
```

**设计亮点**:
- ✅ 专业匹配很重要 (2x生产力)
- ✅ 技能等级有差异
- ✅ 部门之间相互依赖
- ✅ 需要策略分配 (不是简单的"全塞R&D")

#### 维护成本
```
每分钟:
├─ Materials消耗: 员工数 × 1
├─ GMP消耗: 员工数 × 1
└─ 效率惩罚: 未支付 → 50%生产力

风险:
└─ 债务警告: GMP < -200
└─ 游戏结束: GMP < -500
```

**平衡性**:
- ✅ 防止无限扩张
- ✅ 需要平衡收入和支出
- ✅ 创造策略决策 (招还是不招？)

---

### 4. 远征系统

#### 任务设计
```
任务难度 → 战斗力要求
├─ Easy: 需要 2 战斗力
├─ Medium: 需要 3-4 战斗力
└─ Hard: 需要 5+ 战斗力

持续时间 → 45-120秒
└─ R&D 部门可以缩短持续时间

奖励:
├─ Materials, Fuel (资源)
├─ GMP (货币)
└─ Recruits (员工)
```

#### 成功率机制
```
基础成功率: 50%
├─ Intel 部门: +5% × 员工数
└─ 最大: 95%

结果:
├─ 成功 (50%概率): 100% 奖励
├─ 部分成功 (40%): 50% 奖励
└─ 失败 (10%): 0% 奖励 + 可能伤亡

特殊:
└─ 5% 暴击成功: 150% 奖励
```

#### 伤亡机制
```
失败时:
└─ 伤亡率: 15% × (1 - Medical部门加成)

Medical 加成:
├─ 每个员工: -15% 伤亡率
└─ 最多减少到: 0% (6个Medical员工)
```

**设计评价**:
- ✅ 风险回报机制
- ✅ 部门协同效应
- ✅ 惩罚机制 (员工牺牲)
- ✅ 策略深度 (需要平衡部门)

---

### 5. 组合系统 (Combo System)

#### 相邻加成
```
平台 A + 平台 B (相邻)
    ↓
检查组合规则
    ↓
如果有combo:
├─ 资源产量加成
├─ 特殊效果
└─ 视觉反馈
```

#### 示例组合
```
可能的组合设计:
├─ R&D + Support → 研究加速
├─ Combat + Intel → 远征奖励↑
├─ Medical + Any → 恢复速度↑
└─ Support + Support → 床位上限↑
```

**策略价值**:
- ✅ 鼓励多样化布局
- ✅ 增加建造策略
- ✅ 奖励合理规划

├─ 2-5个随机模块

## 🎯 玩家决策点

### 关键决策

#### 1. 优先发展哪个部门？
```
R&D → 技术优势
Combat → 远征优势
Support → 规模优势
Intel → 信息优势
Medical → 稳定优势
```

#### 2. 招募还是扩张？
```
招募员工 → 短期成本，长期收益
建造平台 → 长期投资
```

#### 3. 员工如何分配？
```
专业匹配优先？ → 高效率
还是平均分配？ → 低风险
```

#### 4. 远征风险接受度？
```
高难度高回报 → 可能损失员工
低难度低风险 → 稳定发展
```

#### 5. 平台布局策略？
```
紧凑布局 → 组合加成多
分散布局 → 风险分散
```
