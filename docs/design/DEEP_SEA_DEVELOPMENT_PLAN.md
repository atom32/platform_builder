# 深海飞地 - 游戏开发计划

基于 `clip.txt` 设定的后启示录深海生存模拟

---

## 📖 核心设定回顾

**背景**：核战后42年，太平洋废弃钻井平台
- **坐标**：北纬28°，东经135度
- **初始**：7名幸存者
- **发展**：47人，深海垂直城市
- **深度**：150米海底
- **主题**：生存、开拓、传承

---

### Phase 2: 生存资源层（2-3周）

**目标**：添加生存压力机制

**新增资源**：
```gdscript
// data/core/survival_resources.json
{
  "water": {
    "consumption_per_person": 2.0,  // L/day
    "production": {
      "desalination": 10.0  // L/day from desalinator
    }
  },
  "food": {
    "consumption_per_person": 1.5,  // kg/day
    "production": {
      "hydroponics": 8.0,     // kg/day from greenhouse
      "bioluminescent": 2.0  // kg/day from fungi
    }
  },
  "oxygen": {
    "consumption_per_person": 550,  // L/day
    "production": {
      "electrolysis": 600,    // L/day from generator
      "algae": 50            // L/day from algae
    }
  }
}
```

**新增平台类型**：
```
- 居住舱（Living Quarters）
  - 提供：床位 + 氧气
  - 成本：50 Materials + 20 Fuel

- 水培温室（Hydroponics Greenhouse）
  - 生产：食物
  - 成本：40 Materials + 30 Fuel

- 海水淡化器（Desalinator）
  - 生产：水
  - 成本：60 Materials + 10 Fuel

- 深海发电机（Deep Sea Generator）
  - 生产：电力 + 氧气
  - 成本：80 Materials + 50 Fuel
```

**生存循环**：
```
旧：Materials → 建筑 → 生产 → 扩张
新：Food+Water+O2 < 生存需求 < 生产设施 → 维持/扩张
```

**交付物**：
- ResourceSystem 扩展（水/食物/氧气）
- 新平台类型（4种）
- 生存消耗计时器
- HUD 显示生存资源
- 缺乏警告系统

**优先级**：🔴 高（核心玩法改变）

**机制**：

- 水压影响（深水平台需要加固）

**深海资源**：
```
- 深海鱼类（食用/驯服）
- 热液喷口矿产

- 古代沉船物资
```

**交付物**：
- DepthSystem 单例
- 深度选择UI
- 深海资源采集系统
- 水压计算
- 平台加固机制

**优先级**：🟡 中（扩展玩法深度）

## 📝 核心设计问题

### 问题1: 是否保持实时制？

**实时制（当前）**：
- ✅ 符合"生存压力"的紧张感
- ❌ 需要暂停/加速功能
- ❌ 难以平衡资源消耗

**回合制（之前讨论）**：
- ✅ 易于平衡
- ✅ 玩家有思考时间
- ❌ 可能失去生存紧迫感

**建议**：混合制
- 资源生产：回合制（每回合=1天）
- 生存消耗：实时制（每X秒消耗）
- 可加速/暂停/减速

### 问题2: 死亡惩罚？

**当前**：爬塔失败=员工永久死亡
**clip.txt 暗示**：生存艰难，但充满希望

**建议**：
- 轻度模式：失去资源/装备，员工存活
- 硬核模式：员工死亡，需要招募替代
- 遗产继承：死亡员工的后代有天赋加成

### 问题3: 是否有"赢"？

**clip.txt 暗示**：目标是"活下去"+"传承知识"
**当前系统**：无限扩张

**建议结局**：
1. **生存结局**：坚持X天数
2. **传承结局**：建造"月球纪念碑"，让后代见到月亮
3. **文明结局**：人口达到X，建立深海政府
